## 8. Gap Analysis Reference

A dedicated gap analysis report is available in `docs/corpus_gap_analysis_v1.md`.

Key corpus shortfalls identified:
- Critical: beverages, restaurant meals, diabetic-marketing packaged foods, Indian regional staples, US staple bowls
- High: branded low-GI products, fermented beverages, mixed meal benchmarks, snack bar coverage
- Medium: ingredient diversity in seeds/nuts, pulse variety, packaged meal kits, plant-based dairy alternatives
- Low: seasoning blends, convenience diabetic-friendly snacks, label and marketing variants

M1 expansion should prioritize the highest-impact gaps first and preserve 50/50/50 balance for ingredient/food/product coverage.

## 9. GI Reference Acquisition Plan

### Objective
Acquire scientific GI references for the initial corpus and build a reproducible source acquisition pipeline.

### Steps
1. Prioritize authoritative GI sources: USDA, IFCT, FAO/WHO, academic journal tables, ADA reports.
2. Collect published GI tables for staple ingredients and common meals in India and the US.
3. Capture both measured GI values and contextual metadata (serving size, preparation method, sample population).
4. For products, seek manufacturer nutrient data plus independent GI validations when available.
5. Tag each reference with source type, evidence level, trust score, and date of acquisition.
6. Collect at least 20 high-confidence GI references before expanding to 50 foods/products.

### Expected deliverables
- GI reference catalog with provenance records
- Mapped GI values for initial 50 ingredient and food items
- Evidence entries for each benchmark item
- A repeatable process for adding more references during the growth roadmap

## 10. Corpus Growth Roadmap (50 → 250 → 1000)

### Phase 1: 50 corpus items
- Deliverable: initial 50 Ingredients, 50 Foods, 50 Products
- Focus: core diabetes/prediabetes foods in India and US
- Gate: all items have valid schema, evidence references, and source registry mappings
- Timeline: Sprint 0.5

### Phase 2: 250 corpus items
- Add 100 more ingredients, 100 more foods, 100 more products
- Expand coverage for:
  - legumes and pulses
  - low-GI vegetables and fruits
  - regional Indian meals and US convenience foods
  - beverage and dairy variations
- Improve benchmark depth with 50+ measured or published GI/GL references
- Add more source registry entries (academic journals, regional food authorities, diabetes organizations)
- Gate: 75% of corpus items have A/B evidence; 100% have validated schema and trust metadata

### Phase 3: 1000 corpus items
- Expand broadly across:
  - global ingredient families
  - regional cuisines beyond India/US
  - processed products and functional foods
  - meal kits and common supermarket categories
- Add robust provenance for product barcodes and brand listings
- Expand benchmark set to 200+ validated GI/GL reference items
- Gate: production-ready corpus with explicit diabetes/prediabetes metadata, explainability coverage, and no critical data gaps

## Recommended schema and import design

The package assumes a corpus schema based on the current GlyLens domain model:
- `Ingredient`: canonical ingredient metadata, category, processing level, glycemic profile link
- `Food`: composed food item, ingredient IDs, portions, region, glycemic profile link
- `Product`: packaged item, barcode, brand, ingredient references, profile link
- `GlycemicProfile`: GI value, GL value, impact, confidence, evidence level, source IDs
- `Source`: provenance registry entry with type, trust, and metadata
- `Evidence`: evidence linking entity to source with level, confidence, and provenance

### Import pipeline design
1. Source ingestion
2. Normalization
3. Entity mapping
4. Evidence assignment
5. Validation
6. Benchmark seeding
7. Persistence

### Validation approach
- Validate schema and references
- Validate GI/GL ranges
- Validate evidence and source trust
- Validate regional coverage and product completeness

### Production readiness criteria
- 50/50/50 corpus completed
- India and US packs present
- Source registry seeded
- Benchmark and evidence registry seeded
- QA gates satisfied for schema, data consistency, and provenance
- Documented growth roadmap and acquisition process
